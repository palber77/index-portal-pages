# index-portal-pages

## 项目简介

本项目是一个通用型 HTML 页面归档仓库，用于存放和发布多个独立的静态页面。每个页面以独立文件或子目录形式组织，适合作为页面索引、信息展示或简单工具的集合。仓库不绑定特定域名或网站，仅作为页面文件的托管与版本管理用途。

## 目录说明

```
index-portal-pages/
├── pages/          # 存放各独立 HTML 页面（可包含子目录）
├── assets/         # 公共资源文件（样式、脚本、图片等）
├── index.html      # 可选的主入口页面
└── README.md       # 本文件
```

- `pages/`：每个页面可作为一个 HTML 文件或一个包含 `index.html` 的子文件夹存放于此。
- `assets/`：若多个页面共用样式或脚本，可放置在此目录下，页面内通过相对路径引用。
- `index.html`：可选的根目录入口页，用于列出所有归档页面或提供导航。

## 页面归档说明

- 所有归档页面均为独立、完整的 HTML 文件，无需额外后端支持即可在浏览器中直接打开。
- 页面内容应保持清晰、简洁，避免包含外部追踪代码或动态加载不可控资源。
- 归档页面可通过 GitHub Pages 或其他静态托管服务发布，亦可在本地直接浏览。
- 建议每个页面在 `<head>` 中添加适当的 `<meta>` 标签（如 `charset`、`viewport`）以保证兼容性。

## 维护说明

- 新增页面：在 `pages/` 下新建目录或文件，并更新 `index.html`（如存在）中的链接。
- 更新页面：直接修改对应 HTML 文件，保留版本历史以便回滚。
- 删除页面：移除对应文件或目录，并清理相关引用。
- 提交规范：建议每次提交只涉及一个页面的新增、修改或删除，并附带简要说明。
- 资源管理：公共资源放入 `assets/` 目录，避免在各页面内重复存放。

## 注意事项

- 本仓库不托管任何动态内容或用户数据，所有页面均为静态文件。
- 请勿在页面中嵌入未经授权的第三方资源或违反法律法规的内容。
- 若需通过 GitHub Pages 发布，请在仓库设置中启用 Pages 功能，并选择正确的分支与目录。
